# 项目规则

## 项目定位
这是一个个人主页 + Blog 的 Nuxt 项目，后续会扩展 vibe coding / projects / notes 等模块。
第一版的目标是先完成一个简洁、稳定、可持续扩展的个人站点基础版本。
当前阶段重点是内容展示，不是内容平台，也不是后台驱动系统。

## 技术栈
- Nuxt
- TypeScript
- @nuxt/content
- Tailwind CSS

## 实现原则
- 第一版以静态内容展示为主
- 内容来源优先使用本地 markdown
- 不为了“以后可能会用”而提前引入超出当前范围的基础设施
- 不引入数据库、服务端业务层、后台管理能力
- 不引入与当前目标无关的重型依赖
- 优先保证结构清晰、可维护、可扩展
- 所有设计和实现都必须服务于“个人主页 + Blog 第一版”

## 当前范围
第一版只做：
- 首页
- Blog 列表页
- Blog 详情页
- 标签页
- 主题切换
- 基础 SEO
- 404 页面

## 当前不做
- 数据库
- 登录注册
- 评论系统
- 后台管理
- 点赞/收藏
- 阅读量统计
- 全文搜索

补充说明：
- 当前不单独开发 projects / notes / vibe coding 页面
- 首页不展示这些未来模块的占位或入口
- 不做分页
- 不做归档页
- 不做推荐系统
- 不做复杂筛选
- 不做任何需要持久化用户状态的互动功能

## 代码组织要求
- pages/ 只放路由页面
- components/ 放复用组件
- composables/ 放复用逻辑
- content/posts/ 放 markdown 文章
- public/ 放静态资源

补充要求：
- layouts/ 用于全局布局
- 第一版优先保持目录简单，不提前拆出过多抽象层
- 页面逻辑不要堆积在 components/ 中
- 可复用逻辑再进入 composables/，不要过早抽象

## 内容规范
- 所有 Blog 文章统一放在 content/posts/
- 每篇文章使用一个 markdown 文件
- 文件命名建议与 slug 保持一致，避免路由混乱
- 第一版内容模型保持简单，但 frontmatter 需要保留可扩展性

### 推荐 frontmatter 字段
- title: 文章标题
- description: 文章摘要
- date: 发布时间，格式统一为 YYYY-MM-DD
- tags: 标签列表
- published: 是否发布
- readingTime: 推荐阅读时间
- location: 发布地点

### frontmatter 设计要求
- 以上字段作为当前推荐基础字段
- 不将 frontmatter 结构限制为不可扩展的固定死格式
- 后续允许在不破坏现有结构的前提下继续增加字段
- 第一版实现时应优先兼容可选字段，而不是假设所有字段永远固定不变

### frontmatter 示例
```md
---
title: 文章标题
description: 文章摘要
date: 2026-03-28
tags:
  - nuxt
  - blog
published: true
readingTime: 6 min
location: Shanghai
---
```

## 路由规范
- /：首页
- /blog：Blog 列表页
- /blog/[slug]：Blog 详情页
- /tags/[tag]：标签页
- 404：未命中页面

补充要求：
- Blog 详情页统一基于 slug
- 标签页统一基于标签值生成路由
- Blog 列表默认展示全部已发布文章
- 第一版不加入搜索路由和其他内容频道路由

## 页面要求

### 首页
- 展示个人简介或站点简介
- 提供 Blog 的明确入口
- 可展示最新文章
- 不展示未来模块占位或入口

### Blog 列表页
- 按 date 倒序展示
- 展示标题、日期、摘要、标签
- 保持信息清晰，不做复杂交互

### Blog 详情页
- 展示标题、日期、标签、正文
- 以可读性为第一优先级
- 不加入评论区、阅读量、点赞、收藏等扩展能力
- 支持上一篇 / 下一篇导航
- 上一篇 / 下一篇必须做好边界处理
- 如果不存在上一篇或下一篇，对应入口不展示
- 如果当前只有一篇文章，则整块上一篇 / 下一篇导航不展示

### 标签页
- 展示当前标签下的文章列表
- 展示结构尽量与 Blog 列表页保持一致
- 不做复杂标签云效果

### 404 页面
- 提供明确的未找到提示
- 提供返回首页或 Blog 的入口

## 主题切换
- 第一版支持浅色 / 深色主题
- 支持手动切换
- 建议记住用户上次选择
- 首次进入可以根据时间自动判断默认主题
- 默认规则可使用本地时间，晚于 18:00 自动切换为深色主题
- 主题切换需要有动画，但动画必须克制，不得影响阅读体验或页面性能

## SEO 规则
- 首页、Blog 列表页、Blog 详情页、标签页都应具备基础 SEO
- 每个页面应有明确的 title 和 description
- Blog 详情页应优先支持文章级别的 title 和 description
- Open Graph 不是第一版必做项
- URL 应保持简洁、清晰、可读

## 第一阶段开发顺序
1. 项目基础搭建
2. 全局布局与样式基线
3. 内容模型与文章规范
4. Blog 列表页
5. Blog 详情页
6. 标签页
7. 首页
8. 主题切换
9. 基础 SEO
10. 404 页面

## UI 风格
- 开发者极简风
- 留白优先
- 可读性优先
- 响应式
- 支持浅色/深色主题
- 不要过度动画

补充要求：
- 视觉表达克制，避免花哨和堆砌
- 优先通过排版、间距、层级提升体验
- 移动端优先保证阅读和导航清晰
- 不依赖复杂动效来制造设计感
- 整体风格应偏向简洁、安静、面向内容阅读

## 已确认偏好
- 首页不展示未来模块入口占位
- frontmatter 保持可扩展，当前推荐字段包括 title / description / date / tags / published / readingTime / location
- 需要上一篇 / 下一篇导航，并且必须处理好边界情况
- 主题首次进入可以基于本地时间判断，晚于 18:00 默认深色主题
- 主题切换需要动画，但动画风格必须克制
- SEO 不要求完整 Open Graph 支持
- 暂无指定参考站点、字体或版心宽度，后续按需调整
